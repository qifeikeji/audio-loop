# 音频转发 (AudioLoop)

一个 Android 小应用：把某个音频**输入源**实时转发到某个**输出设备**播放。

支持的输入源：
- 3.5mm 耳机孔（需要能被系统识别为"带麦克风"的线材，见下方说明）
- 手机内置麦克风
- USB 声卡 / USB 耳机
- 蓝牙耳机的麦克风（通话音频 / SCO 通道）

支持的输出目标：
- 蓝牙耳机/音箱（A2DP 高音质通道）
- 3.5mm 有线耳机 / USB 声卡
- 手机内置扬声器

界面上有两个圆角卡片：**输入设备**和**输出设备**，各自是一个下拉选择框。默认都是"自动选择"，也可以手动指定具体设备。选好后点击"开始转发"。转发中卡片会变绿。

## 编译方式（完全通过 GitHub Actions，无需本地环境）

本仓库已经配置好 `.github/workflows/build.yml`：

1. 把本仓库推送到你自己的 GitHub 仓库（`git push`）。
2. GitHub Actions 会自动触发（push 到 `main`/`master`，或手动在 Actions 页面点击 `Run workflow`）。
3. 编译完成后，在对应的 workflow run 页面的 **Artifacts** 里下载 `audio-loop-debug-apk`，解压得到 `app-debug.apk`。
4. 把 apk 传到手机上安装（需要在手机设置里允许"安装未知来源应用"）。

不需要在本地安装 Android Studio / SDK / Gradle，所有编译都在 GitHub 的云端 Runner 上完成。

## 用法一：3.5mm 转蓝牙

1. 蓝牙耳机连接好。
2. 3.5mm 孔插入音频输入线。
3. 打开 App，"输入设备"选"3.5mm 有线输入"（或保持自动），"输出设备"选你的蓝牙设备（或保持自动）。
4. 点击"开始转发"。

## 用法二：两个蓝牙设备（一个当麦克风输入，一个当播放输出）

这是比较高阶的用法，涉及蓝牙音频的两条独立通道：

- **A2DP**：高音质单向通道，只能做"输出"，天生不能当麦克风。
- **SCO / HFP（通话音频）**：双向但只有电话音质（8k/16k 单声道），这是安卓上唯一能把蓝牙设备当"输入"用的方式。

**关键的前提设置**（在手机系统设置里做，不是在本 App 里）：

1. 先把两个蓝牙设备都连接好。
2. 进入手机 `设置 → 蓝牙`，点开设备 A 名称旁的齿轮/设置图标，**只勾选"通话音频"，取消勾选"媒体音频"**。
3. 同样打开设备 B 的设置，**只勾选"媒体音频"，取消勾选"通话音频"**。

这样设置之后，系统才能明确区分"谁是麦克风输入、谁是播放输出"，不然安卓不允许普通 App 去指定"这两个已连接的蓝牙设备里到底该用哪个"，只能依赖系统本身的这个每设备开关。

配置好之后：

1. 打开 App，"输入设备"下拉框里选"蓝牙麦克风 (设备A)"。
2. "输出设备"默认会自动排除掉设备 A，选到设备 B（也可以手动选）。
3. 点击"开始转发"，对着设备 A 的麦克风说话，设备 B 应该能听到。

**已知限制（硬件/协议层面，代码无法绕开）**：

- 走 SCO 的麦克风音质只有电话音质（窄带 8kHz 或宽带 16kHz，单声道），比 A2DP 音乐音质明显差，这是蓝牙 HFP 协议本身的限制。
- 部分手机的蓝牙芯片不支持"同时对两个不同蓝牙设备分别维持 SCO 通话通道 + A2DP 播放通道"，如果尝试后没有声音或者只有一边работает，大概率是芯片并发能力不够，不是软件 bug。
- 建立 SCO 连接通常有 1~2 秒延迟，App 内部有 4 秒超时逻辑，超时会自动放弃本次转发。

## 常见问题：插入普通 AUX 线，"3.5mm 输入"检测不到 / 只能听到自己的声音从 3.5mm 喇叭出来

这是**硬件电气特性**导致的，不是软件 bug：

- 普通"AUX 音频线"是 **3 极 (TRS)**：左声道 / 右声道 / 地。
- 手机耳机孔是 **4 极 (TRRS)**：左声道 / 右声道 / 地 / 麦克风。
- 把 3 极插头插进 4 极插孔时，麦克风触点通常会被短路或悬空，手机检测不到"有效麦克风"，直接判定为"纯耳机输出（无麦克风）"，不会出现在输入设备列表里。

**排查方法**：App 首页有"查看设备详情"按钮，点开会弹出系统当前识别到的所有音频输入/输出设备原始信息。插入线材后点一下，能直接看到系统识别成了什么类型。

**解决方案（任选其一）**：

1. **换用带麦克风识别的四段音频线**：某些直播声卡自带的四段线可以让手机识别为"有线耳机(带麦克风)"。也可以自制一根在麦克风脚和地之间焊接约 2.2kΩ 电阻的转接头，欺骗手机识别电路。
2. **改用 USB 声卡（推荐，最稳定）**：几十元的 USB Type-C 外置声卡，通过 USB-C 口连接，会被识别为标准 USB 音频设备，兼容性远好于耳机孔方案。App 已原生支持识别并使用它。

另外，**之前版本有一个路由 bug**：即使系统识别到了 3.5mm 输入，声音也可能被系统"有线优先"的默认策略又送回 3.5mm 耳机自己的喇叭，而不是送到蓝牙。当前版本已经修复——App 会用 `setPreferredDevice` 显式把播放强制绑定到你在"输出设备"里选择的那个设备，不再依赖系统默认路由。

## 技术要点

- **设备枚举**：`AudioManager.getDevices()` 枚举 3.5mm / USB / 内置 输入输出设备；蓝牙 HFP 设备额外通过 `BluetoothHeadset` Profile 查询"已连接、支持通话音频"的设备（因为 SCO 类型的 `AudioDeviceInfo` 只有在真正建立连接后才会出现）。
- **输入选择**：用户可从下拉框手动选择，或留空走自动优先级（有线/USB > 蓝牙麦克风 > 内置麦克风）。
- **输出选择**：自动模式下会排除掉与输入相同地址的蓝牙设备，优先选另一个 A2DP 设备。
- **强制路由**：`AudioRecord.setPreferredDevice()` / `AudioTrack.setPreferredDevice()` 显式绑定设备，避免系统默认路由策略把声音送错地方。
- **蓝牙麦克风输入**：通过 `AudioManager.startBluetoothSco()` 建立 SCO 通道，监听 `ACTION_SCO_AUDIO_STATE_UPDATED` 广播确认连接建立后，再用 `MediaRecorder.AudioSource.VOICE_COMMUNICATION` 采集。
- **后台存活**：前台服务 + 常驻通知，`foregroundServiceType="microphone"`。
- **目标环境**：`minSdk 28`，`targetSdk 30`（贴合 Android 11 / MIUI 12.5.5），全部界面文案为简体中文。
